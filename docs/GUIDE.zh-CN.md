# B站收藏夹 AI 自动整理 —— 使用指南

> 基于 DeepSeek API，一键智能整理混乱的B站收藏夹

---

## 一、它能做什么

发现 B 站收藏夹越堆越多、想找某个视频翻半天找不到？这个脚本可以：

1. **自动读取**收藏夹中所有视频的标题和简介
2. **AI 语义理解**：不只看关键词，而是理解视频实际内容进行分类
3. **优先匹配已有收藏夹**：尽量把视频归到你已有的分类里，不乱建新分类
4. **自动移动视频**：AI分类后直接帮你把视频挪到对应收藏夹
5. **支持预览模式**：先看AI的分类结果，满意了再执行

## 二、准备工作（3步，约5分钟）

### 第1步：安装 Tampermonkey 扩展

| 浏览器 | 安装地址 |
|--------|----------|
| Chrome | [Chrome 应用商店搜索 Tampermonkey](https://chrome.google.com/webstore/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo) |
| Edge | [Edge 加载项搜索 Tampermonkey](https://microsoftedge.microsoft.com/addons/detail/tampermonkey/iikmkjmpaadaobahmlepeloendndfphd) |
| Firefox | [Firefox 附加组件搜索 Tampermonkey](https://addons.mozilla.org/firefox/addon/tampermonkey/) |

### 第2步：获取 DeepSeek API Key

1. 打开 [platform.deepseek.com](https://platform.deepseek.com/)
2. 注册/登录（支持手机号或微信）
3. 点击左侧「API Keys」→「创建 API Key」
4. 复制生成的 `sk-xxxxxxxx` 密钥（只显示一次，务必保存好）

> **费用说明**：DeepSeek 的 `deepseek-chat` 模型非常便宜，整理几百个视频大约花费几毛钱。新用户通常有免费额度。

### 第3步：安装脚本

1. 点击 Tampermonkey 图标 →「管理面板」→「+」新建脚本
2. 将本目录下的 `bilibili-fav-organizer.user.js` 全部内容复制粘贴进去
3. 找到配置区，把 `apiKey` 改为你的 DeepSeek Key：

```javascript
apiKey: 'sk-xxxxxxxxxxxxxxxx',  // 把你的Key填在这里
```

4. 按 `Ctrl+S` 保存

> 也可以在脚本运行后，直接在弹窗面板中输入 API Key 并点击保存。

## 三、使用方法

### 基础用法

1. 打开 B站，进入任意**具体收藏夹页面**（URL 中带 `?fid=xxx`）
2. 页面左下角会出现粉色 `🤖 AI整理` 浮动按钮
3. 点击按钮，弹出操作面板
4. （可选）在输入框中写整理要求，例如：
   - `前端框架相关统一放「前端」`
   - `分类不要太细，大类即可`
   - `不确定的放「待复核」`
5. 先点 **「👁 仅预览」** 查看 AI 分类结果
6. 确认分类合理后，点 **「🚀 开始整理」** 执行移动

### 整理要求怎么写？（提高准确率的关键）

| 写法 | 效果 |
|------|------|
| 不填 | AI 自由发挥，优先匹配已有收藏夹 |
| `Vue/React相关放「前端」` | 指定某个类别 |
| `分类不要太细` | 减少分类数量 |
| `算法和数据结构的放一起` | 合并相关类别 |
| `不确定的放「待复核」` | 兜底分类，降低误分风险 |

### 预览模式 vs 直接执行

| | 预览模式 | 直接执行 |
|------|----------|----------|
| 调用AI | ✅ 会 | ✅ 会 |
| 移动视频 | ❌ 不执行 | ✅ 执行 |
| 查看分类结果 | ✅ 显示 | ✅ 显示 |
| 用途 | 先看结果再决定 | 一步到位 |

**推荐流程**：先预览一次，确认分类合理，再执行。

## 四、脚本工作原理（流程图）

当你点击「开始整理」后，脚本按以下流程自动运行：

```
┌─────────────────────────────────────────────────┐
│  1. 从Cookie获取登录信息（DedeUserID + bili_jct）│
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  2. 调用B站API，获取你已创建的所有收藏夹列表     │
│     (过滤掉"默认收藏夹")                         │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  3. 分页抓取当前收藏夹内所有视频                  │
│     (每页20条，自动翻页直到取完)                  │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  4. 将视频信息 + 已有收藏夹列表 + 你的要求       │
│     组装成Prompt，发送给 DeepSeek API             │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  5. AI返回分类结果（JSON格式）                    │
│     预览模式 → 显示结果，结束                     │
│     执行模式 → 继续下一步                         │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  6. 遍历分类结果：                               │
│     - 目标已在 → 直接移动                        │
│     - 目标新建 → 创建私密收藏夹 → 移动            │
│     - 目标=源   → 跳过                           │
└────────────────────┬────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────┐
│  7. 完成！显示统计（成功/跳过/失败数量）          │
└─────────────────────────────────────────────────┘
```

### 涉及的 B站 API

| 接口 | 用途 |
|------|------|
| `x/v3/fav/folder/created/list-all` | 获取已有收藏夹 |
| `x/v3/fav/resource/list` | 分页读取视频 |
| `x/v3/fav/folder/add` | 创建新收藏夹 |
| `x/v3/fav/resource/move` | 移动视频 |

### 安全说明

- 脚本**只做移动和新建**，不包含任何删除视频、删除收藏夹的操作
- 新创建的收藏夹默认设为「私密」
- 所有 API 调用都需要 CSRF Token，无法被其他网站伪造
- 脚本仅在 `space.bilibili.com` 域名下运行

## 五、常见问题

### Q1：API Key 报错？
- 检查 Key 格式是否正确（以 `sk-` 开头）
- 检查 DeepSeek 账户余额是否充足
- 确认 Key 没有被泄露/撤销

### Q2：点击按钮没反应？
- 确认你在**具体收藏夹页面**（不是空间首页），URL 中应包含 `?fid=xxx`
- 确认已登录 B站
- 打开 F12 控制台查看错误信息

### Q3：AI 分类不准确？
- 在「整理要求」中写更具体的规则
- 注意你的已有收藏夹名称是否合理（AI 优先匹配已有名称）
- 先用「仅预览」模式检查，调整要求后重试
- 如果某个视频分错了，手动拖到正确分类即可

### Q4：视频太多，处理很慢？
- 每页20条、间隔300ms是B站API的合理频率（太快会被限流）
- 移动操作每批间隔500ms
- 如果收藏夹超过500个视频，建议分批整理

### Q5：能切换其他 AI 模型吗？
可以。修改脚本配置区的 `apiUrl`、`apiKey`、`model`：

```javascript
// OpenAI 示例
apiUrl: 'https://api.openai.com/v1/chat/completions',
apiKey: 'sk-xxx',
model: 'gpt-4o',

// 其他兼容OpenAI格式的API同理
```

### Q6：担心误操作？
- 先用**预览模式**查看结果，不动任何视频
- B站收藏夹也支持手动拖拽移动，错误分类可手动纠正
- 脚本不会删除任何内容
